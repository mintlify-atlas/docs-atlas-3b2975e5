---
title: Repositories
description: Manage Restic repositories for backup storage
---

Repositories are Restic backup storage locations. They can be stored locally, on cloud storage (S3, Azure, GCS), or via rclone-configured remotes.

## List Repositories

<CodeGroup>
```bash cURL
curl -X GET http://localhost:4096/api/v1/repositories \
  -H "Cookie: zerobyte.session=..."
```
</CodeGroup>

Retrieve all configured repositories.

**Response:**

<ResponseField name="repositories" type="array">
  <Expandable>
    <ResponseField name="id" type="string">
      Repository ID
    </ResponseField>
    <ResponseField name="shortId" type="string">
      Short identifier for URLs
    </ResponseField>
    <ResponseField name="name" type="string">
      Repository name
    </ResponseField>
    <ResponseField name="type" type="string">
      Backend type: "local", "s3", "azure", "gcs", "rclone"
    </ResponseField>
    <ResponseField name="config" type="object">
      Backend-specific configuration
    </ResponseField>
    <ResponseField name="compressionMode" type="string | null">
      Compression mode: "auto", "max", "off"
    </ResponseField>
    <ResponseField name="status" type="string | null">
      Repository status: "ready", "locked", "error"
    </ResponseField>
    <ResponseField name="lastChecked" type="number | null">
      Unix timestamp of last check
    </ResponseField>
    <ResponseField name="lastError" type="string | null">
      Last error message if any
    </ResponseField>
    <ResponseField name="doctorResult" type="object | null">
      Results from last doctor operation
    </ResponseField>
    <ResponseField name="createdAt" type="number">
      Unix timestamp in milliseconds
    </ResponseField>
    <ResponseField name="updatedAt" type="number">
      Unix timestamp in milliseconds
    </ResponseField>
  </Expandable>
</ResponseField>

## Create Repository

<CodeGroup>
```bash cURL
curl -X POST http://localhost:4096/api/v1/repositories \
  -H "Cookie: zerobyte.session=..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "S3 Backup Repository",
    "compressionMode": "auto",
    "config": {
      "type": "s3",
      "bucket": "my-backups",
      "endpoint": "s3.amazonaws.com",
      "region": "us-east-1",
      "accessKeyId": "ACCESS_KEY",
      "secretAccessKey": "SECRET_KEY"
    }
  }'
```
</CodeGroup>

Create and initialize a new Restic repository.

**Request Body:**

<ParamField body="name" type="string" required>
  Repository name
</ParamField>

<ParamField body="compressionMode" type="string">
  Compression mode: "auto", "max", or "off"
</ParamField>

<ParamField body="config" type="object" required>
  Backend configuration object (varies by type)
  
  **Local:**
  - `type`: "local"
  - `path`: Local filesystem path
  
  **S3:**
  - `type`: "s3"
  - `bucket`: Bucket name
  - `endpoint`: S3 endpoint
  - `region`: AWS region
  - `accessKeyId`: AWS access key ID
  - `secretAccessKey`: AWS secret access key
  
  **Azure:**
  - `type`: "azure"
  - `accountName`: Azure storage account name
  - `accountKey`: Azure storage account key
  - `container`: Container name
  
  **GCS:**
  - `type`: "gcs"
  - `bucket`: Bucket name
  - `projectId`: GCP project ID
  - `credentials`: Service account JSON credentials
  
  **Rclone:**
  - `type`: "rclone"
  - `remote`: Rclone remote name (from rclone config)
  - `path`: Path within the remote
</ParamField>

**Response:**

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="repository" type="object">
  Created repository basic info
  <Expandable>
    <ResponseField name="id" type="string">
      Repository ID
    </ResponseField>
    <ResponseField name="shortId" type="string">
      Short ID
    </ResponseField>
    <ResponseField name="name" type="string">
      Repository name
    </ResponseField>
  </Expandable>
</ResponseField>
