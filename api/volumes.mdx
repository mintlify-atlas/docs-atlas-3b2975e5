---
title: Volumes
description: Manage backup source volumes (NFS, SMB, WebDAV, SFTP, Local)
---

Volumes represent the data sources to back up. Zerobyte supports multiple backend types including NFS, SMB/CIFS, WebDAV, SFTP, and local directories.

## List Volumes

<CodeGroup>
```bash cURL
curl -X GET http://localhost:4096/api/v1/volumes \
  -H "Cookie: zerobyte.session=..."
```
</CodeGroup>

Retrieve all configured volumes.

**Response:**

<ResponseField name="volumes" type="array">
  <Expandable>
    <ResponseField name="id" type="number">
      Volume database ID
    </ResponseField>
    <ResponseField name="shortId" type="string">
      Short identifier for URLs
    </ResponseField>
    <ResponseField name="name" type="string">
      Volume name
    </ResponseField>
    <ResponseField name="type" type="string">
      Backend type: "nfs", "smb", "webdav", "sftp", or "local"
    </ResponseField>
    <ResponseField name="status" type="string">
      Volume status: "mounted", "unmounted", "error"
    </ResponseField>
    <ResponseField name="lastError" type="string | null">
      Last error message if any
    </ResponseField>
    <ResponseField name="autoRemount" type="boolean">
      Whether to automatically remount on failure
    </ResponseField>
    <ResponseField name="createdAt" type="number">
      Unix timestamp in milliseconds
    </ResponseField>
    <ResponseField name="updatedAt" type="number">
      Unix timestamp in milliseconds
    </ResponseField>
    <ResponseField name="lastHealthCheck" type="number">
      Unix timestamp of last health check
    </ResponseField>
    <ResponseField name="config" type="object">
      Backend-specific configuration (varies by type)
    </ResponseField>
  </Expandable>
</ResponseField>

## Create Volume

<CodeGroup>
```bash cURL
curl -X POST http://localhost:4096/api/v1/volumes \
  -H "Cookie: zerobyte.session=..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My NFS Share",
    "config": {
      "type": "nfs",
      "host": "192.168.1.100",
      "path": "/mnt/backups",
      "version": "4"
    }
  }'
```
</CodeGroup>

Create a new volume.

**Request Body:**

<ParamField body="name" type="string" required>
  Volume name
</ParamField>

<ParamField body="config" type="object" required>
  Backend configuration object (varies by type)
  
  **NFS:**
  - `type`: "nfs"
  - `host`: NFS server hostname or IP
  - `path`: Remote path
  - `version`: NFS version ("3" or "4")
  - `options`: Additional mount options (optional)
  
  **SMB:**
  - `type`: "smb"
  - `host`: SMB server hostname or IP
  - `share`: Share name
  - `username`: Username
  - `password`: Password
  - `domain`: Domain (optional)
  
  **WebDAV:**
  - `type`: "webdav"
  - `url`: WebDAV URL
  - `username`: Username
  - `password`: Password
  
  **SFTP:**
  - `type`: "sftp"
  - `host`: SFTP server hostname or IP
  - `port`: Port (default: 22)
  - `username`: Username
  - `password`: Password (or use `privateKey`)
  - `privateKey`: Private key path (optional)
  
  **Local:**
  - `type`: "local"
  - `path`: Local filesystem path
</ParamField>

**Response:** Returns the created volume object (same structure as list response).

## List Files

<CodeGroup>
```bash cURL
curl -X GET "http://localhost:4096/api/v1/volumes/{shortId}/files?path=/data&offset=0&limit=500" \
  -H "Cookie: zerobyte.session=..."
```
</CodeGroup>

List files and directories in a volume path.

**Parameters:**

<ParamField path="shortId" type="string" required>
  Volume short ID
</ParamField>

<ParamField query="path" type="string">
  Directory path to list (default: root)
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Pagination offset
</ParamField>

<ParamField query="limit" type="integer" default="500">
  Items per page (max: 1000)
</ParamField>

**Response:**

<ResponseField name="files" type="array">
  List of file entries
  <Expandable>
    <ResponseField name="name" type="string">
      File or directory name
    </ResponseField>
    <ResponseField name="path" type="string">
      Full path
    </ResponseField>
    <ResponseField name="type" type="string">
      Entry type: "file" or "directory"
    </ResponseField>
    <ResponseField name="size" type="number | null">
      File size in bytes (null for directories)
    </ResponseField>
    <ResponseField name="modifiedAt" type="number | null">
      Last modified timestamp in milliseconds
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="path" type="string">
  Current directory path
</ResponseField>

<ResponseField name="offset" type="number">
  Current offset
</ResponseField>

<ResponseField name="limit" type="number">
  Current limit
</ResponseField>

<ResponseField name="total" type="number">
  Total number of entries
</ResponseField>

<ResponseField name="hasMore" type="boolean">
  Whether more entries are available
</ResponseField>
