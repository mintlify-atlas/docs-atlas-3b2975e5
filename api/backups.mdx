title: Backups

Backup schedules define automated backup jobs with retention policies, inclusion/exclusion patterns, and cron scheduling.

## List Backup Schedules

<CodeGroup>
```bash cURL
curl -X GET http://localhost:4096/api/v1/backups \
  -H "Cookie: zerobyte.session=..."
```
</CodeGroup>

Retrieve all backup schedules.

**Response:**

<ResponseField name="schedules" type="array">
  <Expandable>
    <ResponseField name="id" type="number">
      Schedule ID
    </ResponseField>
    <ResponseField name="shortId" type="string">
      Short identifier for URLs
    </ResponseField>
    <ResponseField name="name" type="string">
      Schedule name
    </ResponseField>
    <ResponseField name="volumeId" type="number">
      Source volume ID
    </ResponseField>
    <ResponseField name="repositoryId" type="string">
      Target repository ID
    </ResponseField>
    <ResponseField name="enabled" type="boolean">
      Whether schedule is active
    </ResponseField>
    <ResponseField name="cronExpression" type="string">
      Cron expression for scheduling
    </ResponseField>
    <ResponseField name="retentionPolicy" type="object | null">
      Snapshot retention policy
    </ResponseField>
    <ResponseField name="excludePatterns" type="string[] | null">
      File patterns to exclude
    </ResponseField>
    <ResponseField name="excludeIfPresent" type="string[] | null">
      Exclude directories containing these files
    </ResponseField>
    <ResponseField name="includePatterns" type="string[] | null">
      File patterns to include (overrides excludes)
    </ResponseField>
    <ResponseField name="oneFileSystem" type="boolean">
      Don't cross filesystem boundaries
    </ResponseField>
    <ResponseField name="lastBackupAt" type="number | null">
      Unix timestamp of last backup
    </ResponseField>
    <ResponseField name="lastBackupStatus" type="string | null">
      Last backup status: "success", "error", "in_progress", "warning"
    </ResponseField>
    <ResponseField name="lastBackupError" type="string | null">
      Last backup error message
    </ResponseField>
    <ResponseField name="nextBackupAt" type="number | null">
      Unix timestamp of next scheduled backup
    </ResponseField>
    <ResponseField name="createdAt" type="number">
      Unix timestamp in milliseconds
    </ResponseField>
    <ResponseField name="updatedAt" type="number">
      Unix timestamp in milliseconds
    </ResponseField>
    <ResponseField name="volume" type="object">
      Associated volume object
    </ResponseField>
    <ResponseField name="repository" type="object">
      Associated repository object
    </ResponseField>
  </Expandable>
</ResponseField>

## Create Backup Schedule

<CodeGroup>
```bash cURL
curl -X POST http://localhost:4096/api/v1/backups \
  -H "Cookie: zerobyte.session=..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Daily Database Backup",
    "volumeId": "abc123",
    "repositoryId": "xyz789",
    "enabled": true,
    "cronExpression": "0 2 * * *",
    "retentionPolicy": {
      "keepDaily": 7,
      "keepWeekly": 4,
      "keepMonthly": 6
    },
    "excludePatterns": ["*.tmp", "*.cache"],
    "oneFileSystem": true
  }'
```
</CodeGroup>

Create a new backup schedule.

**Request Body:**

<ParamField body="name" type="string" required>
  Schedule name (1-128 characters)
</ParamField>

<ParamField body="volumeId" type="string | number" required>
  Source volume ID or shortId
</ParamField>

<ParamField body="repositoryId" type="string" required>
  Target repository ID or shortId
</ParamField>

<ParamField body="enabled" type="boolean" required>
  Whether schedule is active
</ParamField>

<ParamField body="cronExpression" type="string" required>
  Cron expression (e.g., "0 2 * * *" for 2 AM daily)
</ParamField>

<ParamField body="retentionPolicy" type="object">
  Snapshot retention policy
  <Expandable>
    <ParamField body="keepLast" type="number">
      Keep last N snapshots
    </ParamField>
    <ParamField body="keepHourly" type="number">
      Keep N hourly snapshots
    </ParamField>
    <ParamField body="keepDaily" type="number">
      Keep N daily snapshots
    </ParamField>
    <ParamField body="keepWeekly" type="number">
      Keep N weekly snapshots
    </ParamField>
    <ParamField body="keepMonthly" type="number">
      Keep N monthly snapshots
    </ParamField>
    <ParamField body="keepYearly" type="number">
      Keep N yearly snapshots
    </ParamField>
    <ParamField body="keepWithinDuration" type="string">
      Keep snapshots within duration (e.g., "30d", "1y")
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="excludePatterns" type="string[]">
  File patterns to exclude (glob patterns)
</ParamField>

<ParamField body="excludeIfPresent" type="string[]">
  Exclude directories containing these files
</ParamField>

<ParamField body="includePatterns" type="string[]">
  File patterns to include (overrides excludes)
</ParamField>

<ParamField body="oneFileSystem" type="boolean">
  Don't cross filesystem boundaries
</ParamField>

<ParamField body="tags" type="string[]">
  Tags to apply to snapshots
</ParamField>

**Response:** Returns the created schedule object (without volume/repository nested objects).
